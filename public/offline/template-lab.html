B

Visual Tuning

Whole Label Calibration

All offsets are printer dots. Whole-label scale and offset move text, boxes, lines, QR, barcode origins, and logo graphics together for Template Lab proofing; they affect production only after explicit promotion.

QR

QR data always remains the lot number only. X/Y tune the native ^BQN origin; magnification controls printed module size.

Border Visibility

These toggles hide only the named preview/test border area. Text stays in place, and the Bottom Grid/Footer Row is controlled separately from Color, Material, and Tolling field borders.

Logo

Logo controls apply to the static ^GFA logo where enabled. Upload a higher quality PNG to generate a printer-safe ^GFA asset under the controlled ZPL assets folder.

Current Template Field Geometry

These controls are parsed from the selected production `.template.zpl` file. Boosts multiply the parsed font sizes; X/Y offsets add printer dots to parsed positions.

Selected field: -
Bottom Grid / Footer Row

Bottom Grid/Footer Row controls the full-width row near the bottom of the label. Color Field Border, Tolling Field Border, and Material Type Field Border controls only tune their own field boxes.

Advanced Auto-Fit Metadata

Auto-fit metadata remains supported for dynamic fitted fields. It is saved with the staged geometry and promoted as token-preserving template metadata when helper tokens are present.

Export / Save Details

Browser Draft saves unsaved work in this browser only. Save Lab Profile writes lab/test profile JSON to the configured Template Lab profile file; production remains unchanged. Promote writes the production template and production sidecar profile.
